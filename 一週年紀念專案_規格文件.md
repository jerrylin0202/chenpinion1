## 1. 技術規格

- **形式**:單頁網站(原生 HTML/CSS/JS,無框架、無建置步驟)
- **適配**:以**桌面瀏覽器為主**(目標約 27 吋 / 2K),已加入 RWD 斷點讓平板/筆電/橫向手機也能正常閱讀;直向手機顯示「請橫向」提示
- **音效**:互動音效為程式即時合成(Web Audio,無版權問題),右上角可開關
- **存取**:不設密碼;已加 `noindex` / `robots.txt`,不會被搜尋引擎收錄(低調分享)

## 2. 頁面結構(共 12 頁)

`0 封面` → `1–4 回憶頁` → `5 結尾告白頁` → `6–11 快問快答(6 題)`

導覽規則:
- **下一頁箭頭**只在回憶頁(2–5 頁)出現;封面用「翻開故事」鈕、結尾用信件鈕、快問快答用 A/B 選項前進。
- **上一頁箭頭**除封面外都有。
- 鍵盤方向鍵、觸控滑動與按鈕行為一致(前進僅限回憶頁)。底部頁碼圓點已隱藏。

### 回憶頁版型(每頁不同,文字/角色/照片位置對應變化)

| 頁 | layout | 配置 |
|----|--------|------|
| 回憶1 | `default`   | 角色+文字置中,照片置中於下方 |
| 回憶2 | `stack`     | 照片 2 欄錯落置左,角色+文字置右 |
| 回憶3 | `split`     | 文字置左欄,照片 2×2 置右欄(中間漸層分隔線) |
| 回憶4 | `scrapbook` | 文字+角色靠右,照片 4 欄散排於下方 |

> 詳細規則寫在 `index.html` 開頭的「回憶頁版型規則」註解。

## 3. 已實作功能

- **角色情境舞台**:去背角色放進有機形狀色塊(pond 綠 / blossom 粉 / sunny 黃),帶輕微浮動動畫。
- **拍立得照片牆**:白框 + 手寫小語(Caveat / 粉圓)+ 隨機傾斜 + 圖釘;點擊以燈箱放大。縮圖用 `object-fit:contain` 完整呈現整張照片不裁切。
- **影片**:回憶3 內嵌一支 MP4(格子內靜音循環自動播放,點擊於燈箱放大、可開聲音播放)。
- **插畫貼圖**:每頁角色/文字旁擺去背貼圖,原地浮動動畫;星星✦/愛心♥/氣泡為背景氛圍,永遠在所有內容之後。
- **結尾告白信**:點按鈕以浮動視窗(modal)呈現;信末有「進入快問快答🎁」按鈕。
- **快問快答**:6 題,每題問句 + A/B 兩選項(無對錯,選任一進下一題);最後一題 Q6 點任一選項後,**持續飄下五顏六色的彩帶**。
- **文字格式**:內文支援 `<br>` 換行與 `<span class="hl / hl-teal / hl-gold">` 自訂顏色強調。

## 4. 字體

- 主要字體為 justfont **粉圓體(openhuninn 2.0)**,以 `@font-face` 透過 jsDelivr CDN 載入(webfont,`font-display:swap`),**任何裝置都能顯示**正確字體。
- 備援:Zen Maru Gothic / Noto Sans TC(Google Fonts);手寫小語用 Caveat。
- 設定集中在 CSS `:root` 的 `--font-round` 變數與檔頭 `@font-face`。

## 5. 素材與檔案管線

**命名規則**
- `M01`/`M02`/`M03`/`M04` = 對應回憶 1–4;`STKR-0X` 數字 = 對應回憶頁。
- 照片/影片檔名 `-` 之後的中文 = 在卡片上顯示的小語(影片例外為英文)。

**處理流程(已完成)**
- **照片壓縮**:原圖縮到長邊 1400px、JPEG 品質 82,存為 `*-web.jpg`(含 EXIF 旋轉校正);**原始大檔保留未覆蓋**。
- **貼圖去背**:`STKR-0X.png` 的 AI 生圖假透明棋盤格已用 flood-fill 去背,存為 `STKR-0X-cut.png`(保留白色貼紙外框)。
- **影片**:檔名去空格為 `M03-Good-night-I-Love-You.mp4`(避免主機路徑問題)。
- **角色主視覺**:封面合照與各頁角色圖為 base64 內嵌於 `index.html`,無外部相依。

**版本控管(`.gitignore`)**
- 上傳:`index.html`、`*-web.jpg`、`STKR-0X-cut.png`、影片、workflow、文件。
- 不上傳:原始大照片(`M0*` 未壓縮)、`STKR-0X.png` 原圖、`.claude/`(本機設定)。

## 6. 部署與後續維護(CI/CD)

- GitHub Actions workflow:`.github/workflows/deploy.yml`,push 到 `main` 自動部署到 Pages。
- **修改內容的步驟**:編輯 `index.html` 後,於專案資料夾執行:
  ```
  git add -A
  git commit -m "更新內容"
  git push
  ```
  約 1 分鐘後網站自動更新。

## 7. 隱私說明

- GitHub Pages 免費方案需 repo 公開,故**原始碼/已上傳的照片技術上可被找到 repo 的人下載**;網站本身則靠「不公開網址 + noindex」保持低調。
- 若日後想更私密(repo 不公開),可改用 Cloudflare Pages / Netlify 連私庫部署。
